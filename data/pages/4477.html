As you quoted requests, do what it does and write the index page of the docs, with the common examples of use, and make them easy. Then implement that, and that only. I call that "documentation driven development" or D³.

And it is basically

from fluentmail import mail
letter = mail(from=…, to=…, subject=…, body=…, markup="markdown", format="html+text",
         headers={…}, attachments={…})
letter.send((host, port)) # shortcut for m.send(fluentmail.SMTP(host, port, password=…))

with body taking text + streams, and as you can see an easy way to feed a dual html/text mail. For finer control, accept a dict for body, with html + text keys. send() should default to localhost SMTP.

In addition, you might want to add an actual ‘fluent’ interface, i.e. mail().from(…).to(…)….
