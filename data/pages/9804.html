Description: will-iam - Handle IAM on the commandline

########################################

.. image:: https://travis-ci.org/elbaschid/will-iam.svg?branch=master

:target: https://travis-ci.org/elbaschid/will-iam

Installation

------------

For regular use::

$ pip install https://github.com/elbaschid/will-iam/archive/master.zip

For development::

$ pip install -e ".[dev]"

Using `will.iam`

----------------

You can define some default values in a `william.yml` that either has to be in

a config file in the directory your are running the command in or in

`$HOME/.will.iam.yml`.

.. code:: sh

paths:

vancouver: /my-org/vancouver

chicago: /my-org/chicago

roles:

product:

-

Create a new user

-----------------

Create a user without any policies attached::

$ will.iam user create <user>

Create a user with a specific role::

$ will.iam user create <user> --role <role>

List policies

-------------

List all the policies the account you have access to::

$ will.iam policy list

Export policies

---------------

Export all the existing IAM policies into a subdirectory. Policies are always

stored in a subfolder per region (is IAM region specific???)::

$ will.iam policy export all

Export only policies that can be modified::

$ will.iam policy export custom

Export only policies that can be modified but only in a specific region::

$ will.iam policy export custom --region us-west-1
