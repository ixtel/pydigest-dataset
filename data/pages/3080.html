Yaosf

Yet Another Open Standard Format

I have used JSON, and XML, and YAML, and they all seem somewhat... I don't know. Bloated? I mean the code isn't bloated, it just usually invovles having a module you have to load from somewhere, and make sure it's installed.

SO WHAT THE HELL MAKES YAOSF BETTER?

Well, firstly the format is extremely simple, which is sometimes all we ever need, and secondly the whole function to parse is only 12 lines of python; so you can just drop this in your script/program/amazingmachine/baby/toaster and it will let you whip up an easy lil config file.

Usage????

The Config file format is the following

variable1:"string"
variable2:1234
variable3:1.234

So there is support for strings, integers and floating points.

And to get the data out of it, the function syntax is....

getVar("variable", "config.txt")

This is just return the value of variable in the config file.

So you can print(getVar("variable", "config.txt"), or assign to variable age = getVar("age", "config.txt")
