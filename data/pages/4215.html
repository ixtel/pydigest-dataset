django-keepdb is a Django app that allows you to reuse the same database between 2 test runs. KeepDB has been tested on Django 1.4/1.5/1.6

Usage

  1. Add a new TEST_RUNNER to your django settings like this:

    TEST_RUNNER = 'keepdb.runners.KeepDBTestRunner'

  2. Run python manage.py test to launch the tests.

  3. If it's not your first launch, Django will ask if you want to destroy current test database. Just answer no.

  4. Enjoy.

No input

Django will ask you if you want to destroy current test database before launching again your test suite.

If you want to avoid that, you can use custom KeepDBNoInputTestRunner (only works for now on python 2.7/Django 1.4)

South

If you are using south, to make it work properly with SOUTH_TESTS_MIGRATE option, you need to use custom test command created in keepdb app.

  1. Add keepdb to your INSTALLED_APP Django settings. IMPORTANT: keepdb has to be declared AFTER south app.
  2. Call your Django test command with --south-keepdb option.
