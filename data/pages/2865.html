Introducing ViperPy – A Python library for interacting with the EMC ViPR API

ViperPy is a Python library for interacting with the EMC ViPR API. ViperPy began originally as an easy way to hook up numerous projects that needed to interact with the ViPR product in order to perform certain metering and billing extraction and handle authentication tokens behind the scenes. Very quickly I realized that it could be useful to others in my group if I added support for additional APIs and certain other features. ViperPy gives a great alternative to the existing Java SDK for ViPR if you would prefer to write your application in Python.

Full disclosure: I currently work for EMC in their Rubicon Cloud services group. I in no way speak on behalf of the company or their products.

ViperPy is now being used by our group for production applications and is planned to be used as the basis for our upcoming testing suite against new ViPR releases. Installing ViperPy via PIP is very easy:

$ pip install viperpy

Note: ViperPy currently requires Python 2.7.x. Python 3 support is planned.

You’ll find ViperPy very easy to use and examples can be found in the readme on Github.

There is still plenty of work to be done like adding additional unit tests, additional API support, etc. however the main control plane API calls are already supported. Since we are actively using this library keep on eye on the repo for any changes and enhancements.

One thing to keep in mind is ViperPy is meant to work with the ViPR control plane, not the data plane. For that you can use Python libraries like Boto since ViPR supports many of the Amazon AWS S3 APIs.

Pull requests are always welcome as is good feedback.
