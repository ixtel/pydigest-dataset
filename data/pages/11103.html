.pyi generators

Yes, I guess anyone who wants to use compile-time type checking in Python, probably ends up using MyPy. MyPy contains stubgen.py tool which generates .pyi files.

Usage

(all from docs in stubgen.py file)

mkdir out
stubgen urllib.parse

generates out/urllib/parse.pyi.

You can use it for python2 too:

stubgen --py2 textwrap

or for C modules:

scripts/stubgen --docpath <DIR>/Python-3.4.2/Doc/library curses

This project is dedicated to exactly this goal.

Yes, if you're using .pyi files in your own project, you probably want to use this also when using external code. Typeshed contains .pyi files for Python2 and Python3 stdlib and a bunch of Python2 libraries (like redis, crypto, requests...) and some Python3 libraries (like lxml or requests), all nicely versioned.
