Sinklog combines your log outputs into a single stream. All you need to start logging is a name. It has to be globally unique–so pick a good one!

How To Use It

Use the logger command or another syslog client to send a message. Check out the tools and examples on GitHub. 

logger -t logkey -n sinklog.com "my log message"

Logs are aggregated into a single stream and made available over HTTP so they can be viewed live in a browser or terminal. Itâs like running tail -f across multiple hosts.

curl -ns https://sinklog.com/s/demo-logs-123

How It Works

Logs are transported over syslog and aggregated by tag. After you pick a name you will receive a log key to use as a tag that can be used with your existing loggers.

Log data is not archived, it is kept in a short buffer and flushed. Sinklog is not a log storage service.
