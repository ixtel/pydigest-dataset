    • Release 1.4.3
    • Test selection by attribute

      Confirm that selection by attribute works. Fixes #142.
    • Add Django 1.9 test arguments
    • Convert optparse callback to argparse

      In Django 1.8 and later, handle callback args by converting to
      action=store, and dropping the callback arguments. Django should be
      able to parse the command line without the callback, and then nose will
      parse the command line and call the callback with any given options.

      Fixes #223
    • Hard-code options with arguments for Django 1.7

      For Django 1.7 and earlier, hard-code the list of options that include a
      parameter.

      Fixes #7
    • Handle long options without an equals sign

      When a long option is used without an equals sign (such as --processes 2
      or --verbosity 2), the argument is passed to nose. This only works in
      Django 1.8 or later. For Django 1.7 and earlier, --verbosity is handled,
      but the caller must use --option=value for other parameters.

      Fixes #7
    • Add Django 1.8 test arguments

  • Released at May 3, 2012
    Source: VCS
    • Make FastFixtureTestCase and TransactionTestCaseReorderer get along better.

      I imagined a situation in which the fixture bundler would shove a TransactionTestCase in the middle of a bundle of FFTCs and the TTC wouldn't clean up after itself, thus wrecking the bundle. So I made the bundler much more conservative: now it includes only FFTCs in bundles, which is fine, because nobody else was going to implement my advisory bits anyway.

      Then the test types that concern the bundler and the TTC reorderer are disjunct, making the interaction of the 2 plugins much easier to think about.

      I also narrowed the states returned by the TTC reorderer to 2, since all we really care about are "clean" and "dirty".
