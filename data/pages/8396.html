    • fix error when running on Windows with no ctypes module

      The problem has been encountered using 64-bit CPython 2.5.4 that does not
      include the ctypes module by default.

      There AnsiToWin32 object's win32_calls attribute would get initialized to None
      if ctypes is not available. However, later in AnsiToWin32.call_win32() this
      attribute gets iterated over as if it were a dict. Fix is to initialize it to
      an empty dictionary in case Windows APIs can not get called.

    • fix issue #56 - 64-bit CPython 2.5 does not include ctypes on Windows

      The issue was originally introduced in commit
      8cc97352831ba804cd55eec408b088553e0245d4 which resolved issue #47 (pyreadline
      incompatibility) but forgot to fall back to the original default behaviour in
      case the fix was not supported by the surrounding Python environment.

      This should not cause any new pyreadline incompatibility issues because:
      1. fallback code does not use any Windows structure definitions that might
      conflict with those defined by pyreadline.
      2. pyreadline does not support Python 2.5.
